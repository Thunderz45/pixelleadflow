Single Purpose Statement

help_center What is the single purpose of your extension?
LeadFlow Agent automates the collection of public business listings (such as name, rating, reviews, phone number, address, and website) from Google Maps search results and synchronizes them directly with the user's campaign dashboard.
vpn_key

Permission Justification

database Why does your extension require the "storage" permission?
Required to temporarily cache the active scrape campaign status (current search keyword, location, project selection, and collected count) inside the sandbox. This ensures the scraping state recovers gracefully across page redirects/reloads without data loss.
cookie Why does your extension require the "cookies" permission?
Required to synchronize authentication between the extension and the Next.js Dashboard. The extension reads the leadflow_auth_token cookie to verify the user's active session and authorize syncing lead data to their secure database.
alarm Why does your extension require the "alarms" permission?
Required to schedule a lightweight periodic synchronization check (once every minute) to keep the authentication status updated and synchronized with the dashboard session.
cancel Why does your extension require the "tabs" and "scripting" permissions?
Not Requested / Removed. In alignment with the Principle of Least Privilege, these high-risk permissions have been removed from the extension manifest. General window controls are handled via native runtime APIs and DOM parsing is restricted to static declarative scripts.
link Why does your extension require the requested host permissions?
  • https://www.google.com/maps/* & https://www.google.co.in/maps/*: Required to execute the scraper content script inside Google Maps searches.
  • https://leadflow.pixelstudiox.in/* & https://pixelleadflow.vercel.app/*: Required to coordinate cookie tokens and post collected lead profiles directly to the server database.
security

Remote Code & Data Usage

code_off Are you using remote code?
No. All scripting logic, DOM selectors, and event loops are locally compiled and packaged inside the static extension bundle. No remote evaluation or code fetching is utilized.
list_alt What user data do you plan to collect from users?
Data Category Status Justification
Personally Identifiable Information (PII) No No personal user name, physical address, or profiles are read or exported.
Authentication Information Yes Reads Firebase JWT token from the session cookie purely to authenticate requests to the database.

Website Content Yes Parses publicly visible text nodes from Google Maps (name, rating, phone, address, website).
Health, Financial, Personal Communications, Web History, User Activity No Does not access, read, or process any other category of user data.
